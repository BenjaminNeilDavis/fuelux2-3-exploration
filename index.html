Fuel UX V2 and V3 Playing Nice

Migration from fuel UX V2 to V3 can be a long process. Migration from one to the other could involve both multiple versions of the same library cohabitating. This site contains several alternatives for this possibility.

No Conflict Method

This option involves the developer manually calling the noConflict method contained alongside the constructor for each FuelUX control

Constructor Modification At the Library Level

If this option is chosen. A special version of Fuel UX V3 will have to be used and controls will have to be instantiated without jquery convenience methods.

Isolate Controls/Elements within iframes

This option could be used with larger elements to isolate the context of Fuel UX/Bootstrap V2.

Created as a POC for furthering adoption of the latest version of fuelux
